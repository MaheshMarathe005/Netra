14-Day Task Breakdown
  • Day 1–2
    Design SQLCipher database schema: tables for Personnel (id, name, embedding_blob, enrolled_at), AttendanceLog (id, personnel_id, timestamp, liveness_method, confidence, synced), SyncQueue (id, record_id, type, created_at, retries). Set up encryption key derivation strategy: PBKDF2-SHA256 (100k iterations) seeded from device UUID + app secret — key stored in react-native-keychain (Keystore/Secure Enclave backed).
    Schema Design
  • Day 3–4
    Implement SQLCipher integration via react-native-sqlite-storage (fork with cipher support). Write StorageService class with full TypeScript API: enrollPersonnel(), getEmbedding(), logAttendance(), getPendingSync(), markSynced(), purgeAttendanceLogs(). Write unit tests for each method. Verify AES-256 encryption is active (hexdump of DB file must show no plaintext).
    Local Storage
  • Day 5–6
    🔴 HANDOFF — Expose StorageService as a clean module to P2. Provide integration guide (function signatures, error codes, async patterns). Simultaneously: provision AWS infrastructure — S3 bucket (attendance-records-netra, versioning on, SSE-S3 encryption), Lambda function (netra-sync-handler, Python 3.11), IAM roles, Amplify project init with DataStore model.
    Handoff → P2
  • Day 7–8
    Build Sync Engine: NetInfo listener in React Native — on network CONNECTED, trigger SyncService.syncPendingRecords(). SyncService reads SyncQueue → batches records → POST to API Gateway → Lambda validates + inserts to S3 as JSON per record (key: deviceId/date/recordId.json). On 200 response: mark records synced in local DB. Write retry logic: max 3 retries with exponential backoff.
    Sync Engine
  • Day 9–10
    Build Purge Mechanism: post-sync purge function deletes attendance log rows where synced=1 AND timestamp < (NOW - 24h). Embeddings are NEVER purged locally (only on explicit device de-provisioning). Test full cycle: log 10 records offline → restore network → verify sync → verify purge → verify S3 contains all 10 records. Test idempotency: replay same sync — no duplicates in S3.
    Purge & Safety
  • Day 11–12
    Security hardening: certificate pinning for API Gateway (prevent MITM on sync). Implement jailbreak/root detection (react-native-jail-monkey) — refuse enrollment on compromised devices. Validate embedding blob integrity on read (SHA-256 checksum stored alongside blob). Write security threat model document.
    Security
  • Day 13–14
    End-to-end sync stress test: 500 attendance records, 3 network drops mid-sync, verify zero data loss. Write AWS infrastructure guide (for P4 documentation). Monitor Lambda logs for errors. Final review with P2 on sync status screen integration.
    Docs for P4
